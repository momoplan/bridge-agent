# 企业 Codex 组织版交付文档总览

本文面向平台实施、客户成功、售前解决方案和企业客户管理员，说明 Codex 以企业组织版交付时应随项目一并交付的文档集合。

Codex 安装交付文档按读者拆成三层：

- [企业 Codex 初始用户实施 SOP](./codex-implementation-sop.md)：给平台实施、客户成功、平台运维和企业管理员，包含组织开通、身份映射、自动凭证开通、router/计量/审计校验和验收。
- [Codex 工作区用户安装与使用指南](./codex-user-quickstart.md)：给已经加入工作区的团队成员、试点工程师和初始用户，只包含安装百积木、授权、点击 百积木首页“打开控制台”、在百积木平台 Agent 对话页使用 Codex 设备安装能力安装并打开 Codex App、按需启用终端能力、首次验证和常见问题。
- [百积木个人用户安装指南](./bridge-agent-user-install.md)：给个人用户或第一次连接设备的普通用户，只包含 百积木安装、百积木账号登录、设备授权和一句话触发 Codex 安装。

不要把底层鉴权服务、凭证绑定、模型密钥、router 诊断等内部细节写进个人用户或工作区用户文档。

## 1. 交付目标

企业组织版 Codex 的交付目标不是单机安装一个命令行工具，而是完成一套可运营、可审计、可扩展的组织级 AI 编程能力接入：

- 企业用户可以在授权设备上安装并使用 Codex。
- Codex 请求通过我们的路由、账号、工作区、项目和模型密钥体系进行鉴权与计量。
- 平台可以识别请求属于哪个企业、工作区、用户、设备和项目。
- 管理员可以管理成员、权限、模型额度、设备授权、审计日志和问题排查。
- 实施团队可以按标准流程完成安装、验收、培训和后续升级。

## 2. 核心角色和责任边界

| 角色 | 主要责任 |
| --- | --- |
| 企业管理员 | 提供组织、成员、设备、网络和安全策略信息；确认权限和验收结果。 |
| 初始用户 | 在自己的设备上完成安装、登录、授权和首次验证。通常是客户侧管理员、技术负责人或试点工程师。 |
| 平台实施人员 | 确认组织、工作区、项目、用户和设备映射；指导安装；完成端到端验证。 |
| 平台运营人员 | 维护模型供应、额度、计费、审计、故障处理和版本升级。 |
| 企业安全/IT | 审核网络放行、数据边界、密钥管理、日志留存、终端安装和合规要求。 |

责任边界原则：

- Codex App 和 Codex 终端能力运行在客户授权的用户设备上；App 是普通用户的默认安装对象，终端能力用于企业 router、计量和命令行工作流。
- bridge-agent 负责把客户本机授权能力安全接入平台，不替代客户本机系统权限。
- relay/router 负责转发、鉴权、路由、审计和计量，不直接拥有客户本机执行权限。
- 企业管理员负责组织内成员和设备授权策略。
- 平台方负责系统账号、模型路由、自动凭证生成与绑定、服务可用性和交付验收工具链。

初始用户安装过程需要拆成“实施人员文档”“工作区用户文档”和“个人用户文档”，因为不同读者需要的信息完全不同：

| 文档 | 读者 | 应包含 | 不应包含 |
| --- | --- | --- | --- |
| 实施 SOP | 平台实施、客户成功、平台运维、企业管理员 | 身份映射、组织/工作区/项目准备、自动凭证开通、设备归属、router/计量/审计验证、验收清单 | 面向普通用户的简化话术 |
| 工作区用户安装指南 | 已加入工作区的团队成员、初始用户、试点工程师 | 安装百积木、登录授权、点击“打开控制台”、在百积木平台 Agent 对话页使用 Codex 设备安装能力安装并打开 Codex App、按需启用终端能力、首次验证、常见问题 | lowcode-apikey、模型密钥、底层凭证绑定、router 诊断细节 |
| 个人用户安装指南 | 个人用户、第一次连接设备的普通用户 | 安装百积木、登录百积木账号、授权当前设备、用一句话触发 Codex 设备安装能力 | 企业账号、企业交付、router 诊断、凭证绑定、模型密钥 |

## 4. 企业组织版应交付的文档清单

企业版交付不应只给安装步骤。建议把交付资料分成售前、安全、实施、管理员、用户、运维和验收七类。

### 4.1 售前与方案类

| 文档 | 目标读者 | 内容重点 |
| --- | --- | --- |
| 产品白皮书 | 企业决策人、技术负责人 | 产品能力、典型场景、组织版价值、边界说明。 |
| 企业解决方案说明书 | CIO、研发负责人、架构师 | 组织、工作区、项目、用户、设备、模型路由的整体方案。 |
| 架构说明书 | 架构师、安全团队 | bridge-agent、relay、router、apikey、模型服务、审计和计量链路。 |
| 数据流说明 | 安全、合规、法务 | 用户输入、代码上下文、日志、模型请求、审计数据的流向和存储边界。 |
| 部署形态说明 | IT、架构师 | SaaS、专属实例、私有化、混合部署的差异和适用条件。 |

### 4.2 安全与合规类

| 文档 | 目标读者 | 内容重点 |
| --- | --- | --- |
| 安全白皮书 | 安全团队 | 鉴权、密钥管理、权限模型、传输加密、日志审计、最小权限。 |
| 网络放行清单 | 企业 IT | 域名、端口、协议、用途、是否必须、是否支持代理。 |
| 权限与角色模型 | 管理员、安全团队 | 组织管理员、工作区管理员、普通成员、审计员等角色权限。 |
| 数据留存与删除说明 | 合规、法务 | 日志、审计、模型请求、上传文件、设备记录的留存和删除策略。 |
| 密钥管理规范 | 平台运维、企业管理员 | 用户密钥、系统模型密钥、吊销、轮换、泄露处置。 |
| 安全评审问答 | 售前、安全团队 | 常见安全问题、边界解释、平台不做什么、客户侧责任。 |

## 6. 对外销售时的最小但完整文档集

如果第一阶段不想一次性交付全部文档，至少应准备以下 11 份：

1. 产品白皮书。
2. 企业解决方案说明书。
3. 架构说明书。
4. 安全白皮书。
5. 网络放行清单。
6. 初始用户实施 SOP。
7. 管理员手册。
8. 百积木个人用户安装指南。
9. Codex 工作区用户安装与使用指南。
10. 用户快速开始。
11. 故障处理 Runbook。
12. 验收测试用例与交付验收报告模板。

这 12 份文档覆盖了企业采购最关心的五个问题：能做什么、怎么接入、是否安全、如何管理、如何验收。

## 7. 后续产品化建议

为了把 Codex 从项目制交付变成标准企业组织版产品，建议持续完善以下能力：

- 管理员后台沉淀“一键创建 Codex 初始用户”和批量开通向导，并把凭证生成、模型绑定、router 配置和验收命令串成自动流程。
- 百积木支持 Codex 命令行工具的安装、升级、卸载和健康检查所需的本机执行与授权能力。
- 管理员后台展示 Codex 使用量、活跃用户、失败率和费用。
- 审计日志支持按组织、工作区、项目、用户、设备和时间范围查询。
- 安装完成后自动生成验收报告。
- 给企业 IT 提供标准网络检测脚本。
- 给企业安全提供标准安全问卷附件。
